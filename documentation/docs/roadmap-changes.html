Road Map and Changes

Road Map

The next scheduled Click Framework release (version 1.1) will include the Velocity engine 1.5 release, table column sorting, bug fixes and documentation updates.

 

Changes

Version 1.1 - ?? November 2006
  • Added Table Column sorting support. See Column sortable property.
  • Added Column width property and convenience methods for setting table data and header styles.
  • Added Form clearValues() method.
  • Improved Table paging across very large lists.
  • Fixed NPE in PropertySelect when label is null. This issue was raised by David Frizelle [132].
  • Fixed OgnlException in Column when evaluating null property paths. This issue was raised by David Frizelle [133].
  • Updated OGNL library to version 2.6.9.

Version 1.0 - 3rd October 2006
  • Added Cheat Sheet guide. This feature was raised by Ahmed Mohombe and Ricardo Lecheta.
  • Added Column maxLength property to limit the number characters displayed.
  • Fixed NPE in Table ActionLink pagination. This issue was raised by Ricardo Lecheta [124].
  • Fixed custom ErrorPage not resolving. This issue was raised by Glen Stampoultzis [126].
  • Fixed NumberFormatException in LinkDecorator [127].
  • Fixed NumberFormatException when using Form.onSubmitCheck() with a FileField. This issue was identified by Satoshi Takata [128].
  • Fixed CayenneForm bug with required property set on Checkbox. This issue was identified by David Frizelle [129].
  • Fixed PropertySelect bug where first item displays twice under certain conditions. This issue was identified by David Frizelle [130].
  • Updated and verified Click Framework and Extras codebase against FindBugs.
  • Updated Cayenne library to version 1.2.1.
  • Updated documentation and examples.

  • Added FlashAttribute class providing flash scope session attributes. Once a flash attribute has been accessed from the session it is removed.

  • Added paging support to FormTable control.
  • Added Form startTag() and endTag() methods to support laying out forms manually.
  • Added Form clearErrors() method.
  • Added Page templating support for JSP pages. This issue was raise by Huy Do.
  • Added Taiwan properties file provided by Huang.
  • Added new Click Logo provided by Ahmed Mohombe.
  • Improved Form client side JavaScript validation provided by Naoki Takezoe [115].
  • Updated Column class to escape HTML characters by default. This issue was raised by Glen Stampoultzis [118].
  • Updated HtmlStringBuffer to improve HTML character escaping performance, removing Apache Commons Lang's use.
  • Updated FishEye project view provided by Cenqua.
  • Updated Cayenne library to version 1.2.
  • Updated Hibernate library to version 3.1.3.
  • Fixed CompressionFilter to compress response by default and to automatically set any configured Click charset. These issues were raised by Ricardo Lecheta and Naoki Takezoe [119].
  • Fixed NPE in DateField when setting value to null. This issue was raised by Geoff Hopson [121].
  • Updated documentation and examples.

Version 1.0 RC1 - 31th July 2006
  • Added FormTable data grid control and associated FieldColumn class.

Version 0.21 - 19th July 2006
Final new features release for Click 1.0.
  • Added Page onRender() method. This addition was raised by Stephen Haberman [105].
  • Auto add public Page fields to the model. See the ClickServlet methods initPage() and createVelocityContext() for details. This addition was raised by Stephen Haberman [109, 112].
  • Refactored panel package with BasicPanel renamed Panel and moved into the core control package, and removing the Panel interface. The ListPanel and TabbedPanel classes have been revised to take advantage of the new Panel capabilities.

    Panels are now rendered as $panel rather than #parse($panel)

    Panels can now define Velocity templates on the classpath as well as the web file system.

  • Added PageLink control. This addition was raised by Stephen Haberman [106].
  • Added OGNL support for rendering Column values.
  • Added support for multiple ActionLink parameters, see setParameter() for details.
  • Added Menu isUserInRoles() method.
  • Added Best Practices topic to Click documentation.
  • Fixed Tomcat deployment issue where Tomcat ClassLoader was locking Click JARs preventing application redeployment. Due locking on URL streams, the deployment technique has reverted back to using uniquely named files (pre 0.19). This issue was raised by Doychi [82].
  • Fixed MessagesMap cache permissions issue. This issue was identified by David Felis [99].
  • Fixed Field setFocus() bug. This issue was identified by David Felis [102].
  • Fixed Context session Locale bug. This issue was identified by Steven Van Hoof [107].
  • Fixed Select validation bug when no options are defined. This issue was identified by David Frizelle [110].
  • Fixed DoubleField validation bug with "0" value. This issue was identified by David Frizelle [111].
  • Moved ActionButton control from Click Extras to core controls package.
  • Removed depreciated Context sessionObject methods.
  • Updated documentation and examples.

Version 0.7 - 28th July 2005
  • Added FileField control for uploading files and modified Form control to support "multipart/form-data" requests. See the Click Examples File Upload page for a demonstration [1237942].
  • Added pages Auto Mapping configuration option [1237397]. This feature can automatically map HTML page templates to Page classes so you dont have to declare them manually in the click.xml configuration file.
    This concept was defined by Ahmed Mohombe.

    This concept was defined by Phil Barnes.

Version 0.5 - 1st June 2005
  • Form control encodes the form "action" URL attribute using encodeURL(String) to ensure session ID is set.

  • Added Session Object utility methods to Context, see getSessionObject.
  • Added error message block horizontal alignment proeprty errorsAlign to Form.
  • Updated documentation and examples.

Version 0.4 - May 2005
  • Implemented Page Templating capability. Enabling you to create a standardized look and feel for your web application and greatly reducing the amount of HTML you need to maintain.
  • Improved Error handling. Application exceptions render error source line when Java source available. For example a NullPointerException.
  • Integrated the DHTML Calendar widget by Dyna.com into DateField control.
  • Added Form and Field readonly property. Modified Field subclasses to render their HTML readonly attribute. If the Form is set to readonly all its fields will also be readonly.
  • Added Control getId() method. Modified control classes to render their HTML id attribute.
  • Added Form columns property to render forms in multi column table layout.
  • Added page path property to Velocity context.
  • Fixed SourceViewer missing resource bug which occured on Orion server.
  • Updated documentation and examples.
