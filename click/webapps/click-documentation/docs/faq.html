Why not use FreeMarker instead of Velocity?
The Click framework uses a custom Velocity 1.5 build to provided enhanced error reporting.

Its actually better than the line-precise error reporting you get in Tapesty, as Click will render the source code with the error. For example:

http://click.sourceforge.net/docs/error-npe.html

How can you integrate Click into Spring?
The Spring non-web components can be readily used in Click as they can with any framework.

The Spring Web Framework however is not complimentary to Click. Spring uses its own Servlet for dispatching requests to its own request handlers ??. ?? handlers are the Spring plugin point for integration.

Click also provides its own Servlet for dispatching requests to Pages and their Controls. In Click the Pages and Controls provide the plugin points for integration.
